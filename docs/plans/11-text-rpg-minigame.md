# 11. 텍스트 RPG 미니게임 (가챠 인벤토리 연동)

상태: 기획 완료 / 개발 대기
선행 조건: [04. 가챠 시뮬레이터](04-gacha-simulator.md), [10. 푸시 알림 시스템](10-push-notifications.md)

## 배경

단순히 스케줄을 확인하고 가챠를 돌려보는 것을 넘어, 앱 내 체류 시간(리텐션)과 커뮤니티 참여도를 극대화하기 위해 '방치형 텍스트 RPG' 미니게임을 도입한다. 
가챠 시뮬레이터에서 획득한 캐릭터를 버리지 않고 가상 인벤토리에 보관하며, 이를 활용해 원정대를 꾸려 보상을 얻는 구조를 띤다.

## 핵심 기획 (방치형 원정대 + 커뮤니티 스탯업)

1. **가상 인벤토리:** 
   - 가챠 시뮬레이터 결과창에서 운 좋게 뽑은 5성/4성 캐릭터를 '인벤토리'에 저장할 수 있다.
2. **원정대 파견 (Idle System):** 
   - 인벤토리에 있는 캐릭터 최대 4명을 하나의 '원정대'로 편성하여 탐험(예: 4시간, 8시간 코스)을 보낸다.
   - 탐험 시간이 종료되면 "탐험 결과 로그(텍스트)"와 함께 보상(경험치, 칭호 등)을 획득하며, 푸시 알림(FCM)을 통해 유저의 재접속을 유도한다.
3. **커뮤니티 포인트 연동:** 
   - 유저가 커뮤니티(공략 게시판)에 글을 쓰거나 댓글을 달아 얻은 '활동 포인트'를 소모하여 원정대의 베이스 스탯(공격력, 탐험 성공률 등)을 업그레이드할 수 있다.
   - 이는 RPG 성장을 위해 유저들이 자발적으로 커뮤니티 활동을 활발히 하도록 유도하는 선순환 구조를 만든다.

## 작업 범위

### 백엔드 (Spring Boot)
1. **DB 엔티티 설계:** `UserInventory` (유저 보유 캐릭터), `UserRpgStats` (유저 RPG 스탯 및 포인트), `Expedition` (현재 진행 중인 탐험 기록).
2. **API 엔드포인트:**
   - `POST /api/rpg/inventory`: 가챠 결과에서 캐릭터 저장.
   - `POST /api/rpg/expedition/start`: 탐험 시작 (진행 시간 계산 기록).
   - `POST /api/rpg/expedition/complete`: 탐험 완료 처리 및 보상/로그 텍스트(난수 기반 생성) 지급.
   - `POST /api/rpg/stats/upgrade`: 포인트 소모하여 스탯 강화.
3. **스케줄러 연동:** 탐험 완료 시간이 도래한 `Expedition` 데이터를 스캔하여 10번 기획의 FCM 로직을 통해 "원정대가 귀환했습니다!" 푸시 발송.

### 프론트엔드 / 모바일 (Flutter & React)
1. **가챠 화면 변경:** 결과 렌더링 UI에 '인벤토리에 담기' 플로팅 버튼 추가.
2. **미니게임 대시보드 UI:** 
   - '내 정보(Profile)' 탭 하단이나, 상단 헤더의 별도 아이콘을 통해 RPG 대시보드 진입.
   - 보유 캐릭터 목록 리스트, 파티 편성 슬롯, 강화 UI 배치.
3. **로그 연출:** 원정 결과 확인 시, 고전 텍스트 머드(MUD) 게임처럼 타이핑 효과가 들어가며 전투 로그를 읽어주는 UI 모달 연출.
